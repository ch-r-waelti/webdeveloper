Git

Ist ein Versionierungstool. Aktuell verwenden wir es mit der git shell, ein Unix-angelehntes Kommandofenster.

Wir verwenden in der Schule zusätzlich die Tools

  • atom: als Editor
  • Browser-Sync: Test Web-Server

Git aufsetzen

Nach dem Laden und installieren sind in der Git Shell zwei, drei Dinge bzw. Kommandos abzuarbeiten:

  1. git config --global user.name <"DEIN NAME">
  2. git config --global user.email <"DEINE EMAIL ADDRESSE">
  3. ssh-keygen -t rsa -b 4096 -C <"DEINE EMAIL ADDRESSE">

Der so generierte ssh-Key ist in Git https://github.com/ einzutragen.

Projekt aufsetzen

  1. Projektordner erstellen: mkdir
  2. im Projektordner Kommando 'git init'

    3. Initialisiert Git und erstellt den 'master'

  3. Files editieren mit 'atom .'

    4. Sollen Files in Unterordnern erstellt werden, in der Shell Direktory erstellen und darin mit 'atom .' weitere Files erstellen

  4. Bearbeitete Files in die 'Staging Area' schreiben: git add <file | .>

    5. '.' lädt alle Files in die 'Staging Area'

  5. Files versionieren: git commit -m ""
  7. Testen (mit Browser Sync) im Projektordner: browser-sync start --server

    8. Tipp: Um den Browser Sync nicht immer nochmals starten zu müssen, am besten eine zweite Shell öffnen, im Projektordner positionieren und das Kmmando dort aufrufen; dann muss nur noch refreshed werden

    Beenden: [cntrl][c] und entsprechend quittieren

Wichtige Kommandos

  • git status

    • Listet den Status der Files und Directories

  • git log

    • Listet auf, was gemacht wurde

    Parameter

    • <ganzzahl>: Listet erste Einträge auf
    • <negative ganzzahl>: Listet letzte Einträge auf

  • git tag

    • Einen Committed 'Release' 'etikettieren'; d.h. man kann einen Commit, der einen komischen Schlüssel generiert benennen und ihn so sprechender machen.

    Der Tag-Name sollte in dieser [master]/Entwickler-Instanz eindeutig sein!

    Es können mehrere Tags pro Commit vergeben werden.

    git <"name des Tag"> <Commit-ID>

    • "name des Tag": Sprechender, in dieser Instanz eindeutiger Name
    • Commit-ID ist die ID, die beim 'git commit' angegeben wird. Sie kann mit 'git log' ermittelt werden
