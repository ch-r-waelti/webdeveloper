JavaScript (zum Inhaltsverzeichnis)

Einleitung (zum Inhaltsverzeichnis)

JavaScript wird - im Gegensatz zu PHP - im Browser ausgeführt. Es braucht dazu nicht einmal einen Server. JavaScript geht auf Brendan Eich 1995 zurück.

Der Einfachheit halber wird JavaScript in diesem Script mit JS abgekürzt.

https://developer.mozilla.org/de/docs/Web/JavaScript

Einfaches Beispiel (zum Inhaltsverzeichnis)


    <!DOCTYPE html>
    <html lang="de">
    <head>
        <meta charset="UTF-8">
        <title>Hallo JavaScript</title>
    </head>
    <body>
    <!--
        HTML-Inhalt
    -->
    
      <script scripttype="text/javascript">
          // einzeiliger Kommentar

          alert("hello JavaScript");

          /*
            mehrzeiliger Kommentar
          */
      </script>
    
    </body>
    </html>

Anwendung im HTML(zum Inhaltsverzeichnis)

Das Coding von JS erfolgt direkt im HTML oder aber es kann in einer/mehrerer Datei(en) referenziert werden. Grundsätzlich sind die Tags <script> und </script> zu verwenden:

  • direktes Coding: 
    
<script  type="text/javascript">
    alert("hello world");
</script>
  • Filereferenz: 
    
<script  type="text/javascript" src="script/script.js"></script>

Anmerkung: type="text/javascript" ist nicht zwingend nötig.

WICHTIG: Das JS soll grundsätzlich unmittelbar vor dem Ende des Bodys ( </body>) eingefügt werden. Sonst wird vor dem Laden und Darstellen der Site das JS geladen und der Inhalt erscheint verzögert.

Mehrfachverzweigung - am besten erklärt mit Beispiel:

'break;' veranlasst JS, den 'switch' zu verlassen - ansonsten würde weiter geprüft, was mitunter erwünscht ist.

try / catch / finally

Versuche etwas in 'try'... falls Fehler (oder ein throw ausgeloest), wird 'catch' ausgeführt. Anschliessend wird - sofern codiert - 'finally' ausgeführt (PHP kennt das nicht!)


  try {
      // versuche etwas
  }
  catch ($fehlerParameter){
      // im Fehlerfall landen wir hier und können (re-) agieren
      // '&fehlerParameter' ist individuell ...
  }
  finally {
      // abschliessende statements
  }
while / do ... while

Tue etwas bis ... widerum ein Beispiel:

Kopfgesteuert:


  while (i != 3) {
      i = 3;  // jetzt wird die Schlaufe abgebrochen
  }

Fussgesteuert:


  do {
      i = 3;  // jetzt wird die Schlaufe abgebrochen
  } while ($i != 3)

Variablen (zum Inhaltsverzeichnis)

In JS werden die Variablen mit dem Schlüsselwort 'var' deklariert. Würde man das 'var', wäre sie global. Die Variablen sind bis zur Zuweisung, welche auch im 'var'-Statement erfolgen kann, neutral. Der Typ wird erst durch die Zuweisung spezifiziert.

So kann es sein, dass eine bisher nummerische Variable zu einem String wird, wenn diese mit mit einem String konkateniert wird!

Eine variable kann verschiedene Werte in ihrem Life-Cycle annehmen - wobei sich das nachstehende nicht nachvollziehen liess - es scheint, dass sie eher synonym sind ...:

  • undefined: Der deklarierten Variablen wurde noch nie ein Wert zugewiesen.
  • null: Die Variable ist deklariert, aber hat keinen Wert (entspricht 'undefined' oder ihr wurde explizit 'null' zugewiesen.

Arrays sind auch eine Art (Standard-) Objekt und verfügen damit auch über einige Methoden:

array.concat(array) Gibt einen Array zurück, welcher den ersten Array (Methodenaufrufer) mit dem zweiten (Argument) verbunden zurückgibt.
array.includes(wert [, startIndex]) Gibt WAHR zurück, ist der Wert im Array enthalten - andernfalls FALSCH.
array.indexOf(wert [, startIndex])

Gibt den ERSTEN Index zurück, wird der Wert im Array gefunden - andernfalls -1. Es kann mit 'startindex' Der - eben - Index, ab welchem gesucht werden soll, angegeben werden.

Wichtig: Als Vergleichsmethode wird '===' angenommen, also der Wert und der Typ müssen stimmen!
array.lengthAnzahl Elemente im Array.
array.pop() Letztes Element eines Arrays entfernen. Als Resultat wird der Wert des 'gepoopten' Elments zurückgegeben. .

array.sort([vergleichsfunktion]) Sortiert einen Array. Standardmässig, also ohne Vergleichsfunktion wird aufsteigend sortiert. Der Funktion werden zwei Paremeter mitgegeben. Der Sort nimmt dafür zwei zu vergleichende Array- Elemente an.
  • Ist das Reusltat negativ, ist die Reichenfolge: a, b
  • Ist das Resultat positiv, ist die Reihenflge b, a
  • Ist das Resultat identisch, bleibt die Reihenfolge der beiden Elmente identisch.
array.splice(elementStart, elementAnzahl [, elementNeu1 ...]) Löscht oder fügt (je nach 'elementAnzahl' und 'elementNeuX' Elemente ab (inklusive) 'elementStart'. Ist zum Beispiel 'elementAnzahl' = 0 und neue(s) Element angegeben, wird das neue Element nach Start 'elementStart' eingefügt. ist 'elmentAnzahl' > 0, wird die Anzahl Elemente (mit 'elementStart') entfernt. Es kann auch gleichzeitig angefügt werden.
Beispiel: array.splice(3, 1, 'neuesItem') gibt einen Array zurück, welcher 'neuesItem anstelle des 4. Elmentes einfügt (1. Elment = Index 0).

Anonyme Funktionen sind Funktionen, die nur einmal definiert werden und auch nur einmal referenziert werden. Sie sind eigentlich nicht wirklich nützlich - ausser evt. im Zusammenhang mit Callback Functions.

Callback Function (zum Inhaltsverzeichnis)

Callback Functions sind Funktionen, die einer Funktion als Parameter mitgegeben werden und in dieser Funktion dann auch verwendet werden.

Diese werden oft verwendet. Beim Event-Handling beispielsweise. Oder speziel im Zusammenhang mit asynchronen Prozessen wie AJAX / XMLHttpRequest.

Triviales Beispiel



    // Definition der Callback Function
    function callbackFunction(arg1) {
        console.log("Argument: ", arg1);
    }

    // Definition der Function, welche Callback als Argument erhält und benutzt
    function usingCallbackFunction(callbackFunctionAsArg) {
        callbackFunctionAsArg("Das soll via callbackFunction ausgegeben werden!");
    }

    // Ausfuehren der Funktion, welche die Callback Funktion ausführt
    usingCallbackFunction(callbackFunction);

AJAX / XMLHttpRequest (zum Inhaltsverzeichnis)

AJAX bzw. XMLHTTPRequest sind Methoden, um mit dem Server Daten auszutauschen, ohne dass die ganze Seite konstruiert werden muss. Wobei AJAX ein Begriff aus jQuery ist, aber grundsätzlich auf JavaScript und XMLHttpRequest basiert.

Die Idee ist, dass man eine HTTP-Adresse aufruft (also zum Beispiel ein php-File, welches auf den Aufruf zugeschnitten ist) und wartet auf das Resultat. Das Resultat wird dann wie mit dem Server vereinbart interpretiert in JavaScript (und DOM-Manipulations). Die Vereinbarung erfolgt natürlich zwischen dem Client-Entwickler und dem Backend-Entwickler statt.

Häufig werden als Datenstrukturen JSON-Formate verwendet - was eine minimale Normierung der Schnittstelle bedeutet (allerdings kommt es auf den Inhalt an und der muss ja dann doch wieder vereinbart werden ...).

Besonders Wichtig ist das Wissen: Es handelt sich um eine asynchrone Funktion, das heisst, der Code läuft weiter (sofern vorhanden) und wartet nicht auf die Antwort - ausser es ist z.Bsp. mit einer sogenannten Callback-Function abgefangen (siehe dazu folgende Beispiele).

Beispiel:



    // Parameter
    var param = {
        // Adresse für Daten
        urlDestination: "https://jsonplaceholder.typicode.com/todos"
    };

    // Variablen
    var result = {};

    // Wird aufgerufen, wenn HTML geladen
    function init() {

        // button
        // <button> on-click-event hinzufuegen; funktion doIt()
        document.getElementById("doIt").addEventListener("click", doIt_clicked);
    }

    // Aktion auf Button
    function doIt_clicked(ev) {
        ev.preventDefault();
        console.log("url in click: ", param.urlDestination);
        callAJAXByJS(param.urlDestination, onDoneAjax);

    }

    // callback-Function, wenn Daten erhalten
    function onDoneAjax(result) {

        console.log("resultat: ", result);

    }

    // Aufruf XMLHttpRequest
    function callAJAXByJS(reqestedURL, onDoneAjax) {

        var xhr = new XMLHttpRequest();

        console.log("doin'  ");

        xhr.open("get", reqestedURL , true);

        // Define function
        xhr.onreadystatechange = function () {
            var status;
            var data;
            if (xhr.readyState == 4) { // done
                status = xhr.status;
                if (status == 200) {
                    data = JSON.parse(xhr.responseText);
                    onDoneAjax(data);
                }
                else {
                    console.log("Error on AJAXJS - Status: ", status);
                }

            }
            else {
                console.log("xhr.readyState: ", xhr.readyState)
            }
        };
        // ... and: ACTION!
        xhr.send();
    }

    // warte bis HTML geladen und dann function init()
    document.addEventListener('DOMContentLoaded', init);
